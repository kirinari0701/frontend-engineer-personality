> Opinionated Codex personality for frontend-heavy work.

## What it does

- Keeps Codex output direct, concise, and execution-oriented
- Standardizes workflow, release, and communication rules
- Gives new workspaces a ready-to-copy personality snapshot

## Quick start

1. Copy `agents.md` into your own Codex personality config.
2. If your workspace uses `AGENTS.md`, paste the same content there as a local snapshot.
3. Start working with the same conventions in place.

## Sync

- Edit `~/.codex/agents.md`
- Commit the change in this repo
- Push it to GitHub
- Copy the updated content into any workspace that should use it

## Files

- `agents.md`: canonical personality content
- `README.md`: usage and sync notes

## Notes

- Workspace `AGENTS.md` files are local copies, not live links
- Keep project-specific overrides in the workspace, not in this shared file
